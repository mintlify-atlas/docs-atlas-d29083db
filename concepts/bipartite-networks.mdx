## Overview

The simulation framework models **bipartite networks** with:
- **Exporters** (M nodes, top tier)
- **Importers** (N nodes, bottom tier)
- **Directed edges** from exporters to importers with positive weights
- **Quota constraints** on importers (must receive ≥ θ fraction of total inflow)

<Info>
**Bipartite property**: Edges only connect exporters to importers, never within the same tier.
</Info>

## Network Generation

From source code (lines 231-238):

```python
exporters = np.array([f"Exp_{i}" for i in range(n_exporters)])
importers = np.array([f"Imp_{j}" for j in range(n_importers)])

def generate_bipartite_edges(dist_name: str) -> pd.DataFrame:
    rows = []
    for e in exporters:
        mask = rng.random(n_importers) < p_edge
        chosen = importers[mask]
        w = sample_weights(dist_name, chosen.size, rng)
        rows.extend([(e, imp, float(wi)) for imp, wi in zip(chosen, w)])
    return pd.DataFrame(rows, columns=["exp", "imp", "weight"])
```

**Generation process**:
1. For each exporter, independently sample each potential edge with probability `p_edge`
2. For selected edges, sample weights from the specified distribution
3. Return DataFrame with columns: `exp`, `imp`, `weight`

## Edge Probability Parameter

The parameter `p_edge` (often called `p` in the code) controls network density.

### Expected Degree

For an importer $j$, the expected in-degree is:

$$
\mathbb{E}[d_j] = M \cdot p
$$

where M is the number of exporters.

### Two Strategies

From lines 460-469:

#### Strategy A: Fixed p (commented out)
```python
# Option A: fixed p across all scenarios (old behavior)
# p_edge_base = 0.10

```

#### Strategy B: Constant Expected Degree (active)
```python
# Option B: adjust p to keep expected target in-degree E[d_j]=Mp constant
target_degree = 20
def p_rule(M, N):
    p = target_degree / M
    return float(min(max(p, 0.0), 1.0))
```

<Note>
**Active configuration**: The framework uses Strategy B to maintain constant expected importer in-degree (20 predecessors) across scenarios with varying M.
</Note>

## Quota Constraints

Each importer $j$ has an implicit quota requirement:

$$
\text{received}(j) = T_j = \sum_{i} w_{ij} \geq \theta \cdot T_j^{\text{target}}
$$

In the simulation:
- **θ = 0.70** (70% quota threshold)
- An exporter is pivotal if removing it drops an importer below 70% of total inflow

### Example

If importer $j$ receives total inflow $T_j = 1000$:
- Threshold: $(1 - 0.70) \cdot 1000 = 300$
- Exporter $i$ is pivotal if $w_{ij} > 300$

## Edge Weight Heterogeneity

Combining bipartite structure with different [weight distributions](/concepts/weight-distributions) creates distinct regimes:

### Gaussian Weights
- Homogeneous flow (all edges similar magnitude)
- No pivotal nodes emerge
- Highly resilient networks

### Student-t Weights  
- Occasional large flows
- Rare pivotal events
- Moderate concentration

### Power-law Weights
- Star-like patterns (hubs)
- Consistent pivotal nodes
- High structural inequality
