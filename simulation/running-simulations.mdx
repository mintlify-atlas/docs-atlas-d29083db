## Multi-Scenario Simulations

The main simulation block explores how pivotality emerges as network imbalance varies.

### Scenario Definition

Define a list of (M, N) configurations to explore:

```python
scenarios = [
    (140, 60),   # Exporter-heavy
    (130, 70),
    (120, 80),
    (110, 90),
    (100, 100),  # Balanced
    (90, 110),
    (80, 120),
    (70, 130),
    (60, 140),   # Importer-heavy
]
```

### Edge Probability Strategy

You can choose between two strategies for setting `p_edge`:

**Option A: Fixed p across scenarios**
```python
p_edge_base = 0.10
def p_rule(M, N):
    return p_edge_base
```

**Option B: Adaptive p to maintain constant expected in-degree** (recommended)
```python
target_degree = 20
def p_rule(M, N):
    p = target_degree / M
    return float(min(max(p, 0.0), 1.0))
```

Option B ensures that the expected number of edges per importer remains constant (E[d_j] = M × p = 20), isolating the effect of network imbalance.

### Full Simulation Loop

```python
# Configuration
theta_param = 0.70
R_param = 100
max_preds_param = 12
seed_param = 42

# Scenarios
scenarios = [
    (140, 60), (130, 70), (120, 80), (110, 90), (100, 100),
    (90, 110), (80, 120), (70, 130), (60, 140),
]

# Edge probability rule
target_degree = 20
def p_rule(M, N):
    p = target_degree / M
    return float(min(max(p, 0.0), 1.0))

# Output directory
base_output_dir = "simulation_outputs"
os.makedirs(base_output_dir, exist_ok=True)

# Storage for plots across scenarios
all_scenario_plot_paths = {
    "P1_ccdf": [],
    "max_P1": [],
    "frac_P1_zero": [],
    "P2_ccdf": [],
    "P1_P2_scatter": []
}

print("Running simulations for multiple scenarios...")
for M, N in scenarios:
    p_edge_param = p_rule(M, N)
    scenario_name = f"exp{M}_imp{N}_p{p_edge_param:.3f}"
    print(f"\n--- Running scenario: {scenario_name} ---")
    
    # Run simulation
    nodes_df, graphs_df = run_simulation(
        n_exporters=M,
        n_importers=N,
        p_edge=p_edge_param,
        theta=theta_param,
        R=R_param,
        max_preds=max_preds_param,
        seed=seed_param
    )
    
    # Generate plots
    current_scenario_plot_paths = make_plots(
        nodes_df,
        graphs_df,
        R=R_param,
        theta=theta_param,
        output_dir=base_output_dir,
        scenario_name=scenario_name
    )
    
    # Collect plot paths for GIF creation
    for plot_type, path in current_scenario_plot_paths.items():
        all_scenario_plot_paths[plot_type].append(path)
    
    # Print summary statistics
    graphs_means = graphs_df.groupby("dist")[[
        "max_P1", "frac_P1_zero", "gini_P1", "top5_share_P1",
        "max_P2", "frac_P2_zero", "gini_P2", "top5_share_P2",
        "gini_P2_pos", "top5_share_P2_pos", "n_P2_pos",
        "mean_targets", "edges"
    ]].mean()
    
    print("\nGraph-level summary means (across replications):")
    print(graphs_means)
```

## Expected Runtime

The notebook output shows the full 9-scenario simulation completes in a reasonable time:

- **Per scenario**: ~10-30 seconds (depends on M, N, R)
- **Full run (9 scenarios, R=100)**: A few minutes

Computational cost scales primarily with:
- Number of edges (M × N × p_edge)
- Number of replications (R)
- `max_preds` parameter (affects P2 computation)
